# 「造句肉鸽」游戏概念研究报告

> 研究日期：2026-03-16
> 核心概念：以中文造句为核心机制的 Roguelike 卡牌游戏

---

## 一、中文句法结构分析

### 1.1 基本句型模式

中文句法与英语有本质区别，这些差异恰好为卡牌设计提供了丰富的设计空间：

| 句型 | 结构 | 示例 | 游戏设计价值 |
|------|------|------|-------------|
| **SVO 基本句** | 主语 + 谓语 + 宾语 | 我吃饭 | 入门套牌，最基础的"攻击" |
| **主题-评论句** | 话题 + 评述 | 这件事，我不同意 | 话题卡可修饰任何句子，增加伤害 |
| **连动句** | S + V1 + V2 | 他跑去买菜 | 多动词连击，连招机制 |
| **兼语句** | S + V1 + O1(=S2) + V2 | 老师让我回答 | 高级句型，需要精确配合 |
| **把字句** | S + 把 + O + V + 补 | 他把书放在桌上 | 强制位移效果，特殊句型卡 |
| **被字句** | S + 被 + (施事) + V | 花瓶被打碎了 | 防御/反击句型 |
| **存现句** | 处所 + V + 名 | 桌上放着一本书 | 场景描述，环境buff |
| **比较句** | A + 比 + B + Adj | 他比我高 | 对比伤害，越悬殊越强 |
| **"是…的"句** | 是 + 信息焦点 + 的 | 他是昨天来的 | 强调句，暴击机制 |

### 1.2 句子成分 → 卡牌分类

**核心卡牌类型设计：**

| 成分 | 卡牌颜色建议 | 功能 | 稀有度分布 |
|------|-------------|------|-----------|
| **主语（名词/代词）** | 🔵 蓝 | 句子的发起者，决定攻击基础值 | 常见→稀有（"我"→"苍穹"） |
| **谓语（动词）** | 🔴 红 | 核心攻击动作 | 常见→传说（"走"→"席卷"） |
| **宾语（名词）** | 🟢 绿 | 攻击目标/效果 | 常见→稀有 |
| **定语（形容词/的）** | 🟡 黄 | 修饰主语/宾语，增加属性 | 常见→稀有 |
| **状语（副词/介词短语）** | 🟣 紫 | 修饰动词，增加效果 | 较少→传说 |
| **补语（结果/程度/趋向）** | 🟠 橙 | 动词后置加成，决定效果强度 | 较少→传说 |
| **量词** | ⚪ 白 | 数量词搭配，正确搭配加分 | 常见 |
| **虚词（的、了、着、过、吗）** | ⚫ 灰 | 语法胶水，激活时态/语气系统 | 常见但关键 |
| **连词（而且、虽然…但是）** | 🔶 金 | 连接多个句子，组合技 | 稀有→传说 |

### 1.3 "好句子"的评判标准 → 评分体系

#### 第一层：语法正确性（基础分 1-10）

- **成分完整**：有主有谓 = 基础合法（+3）
- **搭配正确**：动宾搭配合理（"吃饭" ✓ "吃桌子" ✗）（+2-4）
- **语序正确**：状语在谓语前、补语在谓语后等（+1-3）
- **虚词使用**：了/着/过 用对时态（+1-2）

**搭配合理性是核心难点。** 游戏中可以用预定义的词语搭配表（collocations）+ 规则引擎判定。例如：
- 动词"喝"的合法宾语集合：{水、茶、酒、咖啡、汤、西北风…}
- "西北风"是特殊搭配 → 触发成语/俗语彩蛋

#### 第二层：修辞加成（乘数 ×1.0 - ×3.0）

| 修辞手法 | 检测方式 | 乘数 |
|----------|---------|------|
| **对仗/对偶** | 上下句字数相等 + 词性对应 | ×1.5 |
| **排比** | 三个以上结构相同的分句 | ×1.8 |
| **押韵** | 句末字韵母相同 | ×1.3 |
| **成语使用** | 命中成语词典 | ×1.5-2.0 |
| **双关** | 一词多义同时成立 | ×2.0（难以自动检测，可人工标记特定组合） |
| **夸张** | 特定词汇组合（"三千丈"类） | ×1.3 |
| **比喻** | "如/像/似" + 名词 | ×1.2 |

#### 第三层：主题相关性（上下文加成 +0-50%）

- 当前战斗场景有"主题词"（如：火、水、山、情）
- 句子中包含相关意象 → 额外加成
- 这鼓励玩家根据场景调整"build"

### 1.4 特殊中文语言现象 → 游戏机制

- **一词多义**：同一个字在不同位置有不同功能（"花"= 名词/动词/形容词）→ 万能卡牌
- **四字格**：中文偏好四字结构 → 四张卡凑成四字格有额外奖励
- **语气词系统**："吗"变疑问（控制技能）、"吧"变祈使（buff）、"啊"变感叹（暴击）
- **量词搭配**：正确的量词搭配（一匹马 ✓ 一个马 △）→ 小加分
- **成语拆解与重组**：把成语拆成四张单字卡，集齐触发超级技能

## 二、现有语言/造句类游戏研究

### 2.1 桌游/卡牌游戏

#### Once Upon a Time（很久很久以前）
- **机制**：玩家手持故事元素卡（角色、地点、事件、物品、特征），轮流讲故事消耗手牌
- **判定**：社交判定——其他玩家可以打断（interrupt card）
- **借鉴**：卡牌驱动叙事的核心框架；但判定太主观，不适合单人肉鸽
- **可借鉴点**：卡牌分类方式（元素类型），用完手牌的目标设定

#### Dixit（妙语说书人）
- **机制**：出牌 + 给线索 + 投票猜测
- **判定**：同伴投票，不能太明显也不能太隐晦
- **借鉴**：对单人肉鸽价值不大，但"模糊匹配"概念可用于主题判定

#### Poetry for Neanderthals（穴居人的诗）
- **机制**：只能用单音节词解释目标词
- **判定**：队友猜对 = 得分，用多音节词 = 被充气棒打
- **借鉴**：语言约束 = 乐趣来源。可以设计"只能用X类卡"的挑战关卡

#### Bring Your Own Book
- **机制**：从自带的书里找句子来回答prompt
- **判定**：玩家投票最佳答案
- **借鉴**："从有限资源中组合最佳答案"是核心共鸣——和手牌造句几乎同构

#### Mad Libs（疯狂填词）
- **机制**：预设句子模板，玩家填入指定词性的词
- **判定**：搞笑程度（社交）
- **借鉴**：**最直接的参考**。模板 = 句型卡，填入 = 打牌。但 Mad Libs 是填空，我们要更自由

#### Paperback / Hardback（文字构建桌游）
- **机制**：拼写单词的 deckbuilder——每张卡是一个字母，拼出更长/更难的词得更多分
- **判定**：词典验证
- **借鉴**：**最接近的已有设计**。证明"语言 + deckbuilder"可行。但他们是英文拼词，我们是中文造句

### 2.2 中文语言游戏

| 游戏 | 机制 | 评判 |
|------|------|------|
| **成语接龙** | 用上一个成语尾字作下一个成语首字 | 规则判定（成语词典） |
| **飞花令** | 诗词中包含指定字 | 规则 + 文化知识 |
| **对联** | 给上联对下联，要求平仄、词性、意境 | 规则 + 主观美感 |
| **诗词大会类** | 填字、背诵、理解 | 标准答案 |

这些传统游戏证明：**中文玩家对语言游戏有强烈的文化认同和参与热情**，尤其是涉及诗词成语的。

### 2.3 电子游戏中的文字/语言机制

| 游戏 | 类型 | 语言机制 |
|------|------|---------|
| **Scribblenauts** | 动作解谜 | 输入任何名词，游戏生成对应物体 |
| **Baba Is You** | 解谜 | 推动单词改变游戏规则（"BABA IS YOU" → "WALL IS YOU"） |
| **Inscryption** | 肉鸽卡牌 | 不直接语言相关，但"卡牌 + 叙事 + meta"框架极好 |
| **Slay the Spire** | 肉鸽卡牌 | deckbuilder 的标杆——协同效应(synergy)设计 |
| **Balatro** | 肉鸽 | 扑克牌 + 修饰符 → 组合乘数。**评分系统极度相关** |
| **Bookworm Adventures** | RPG + 拼词 | 拼英文单词攻击怪物，词越长伤害越高 |
| **文字游戏（中文独立游戏）** | 解谜 | 拆解汉字偏旁部首解谜 |
| **最强蜗牛/文曲星** | RPG | 答题/语文知识作为战斗方式 |

**关键发现：Balatro 的评分模型最值得借鉴。** 
- Balatro 中：基础牌面分 × 倍率 = 最终得分
- 我们可以做：基础语法分 × 修辞倍率 × 主题倍率 = 最终伤害

### 2.4 评判方式总结

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **规则引擎** | 确定性、快速 | 难以覆盖所有情况 | 语法检查、成语匹配 |
| **AI 评分** | 灵活、能判断语义 | 延迟、成本、不确定性 | 文学性评分、搭配合理性 |
| **同伴投票** | 社交性强 | 不适合单人 | 多人模式 |
| **预定义组合表** | 可控、可平衡 | 内容量大 | 特殊组合奖励（成语、典故） |

**推荐方案：混合制**
- 语法合法性 → 规则引擎（预定义搭配表 + 句型模板）
- 特殊组合 → 预定义奖励表（成语、对仗、诗句）
- 可选：AI 评审模式（联网时用 LLM 评分，增加不可预测的惊喜）

## 三、游戏设计分析

### 3.1 核心游戏循环

```
战斗开始 → 抽卡（获得词语卡）→ 手牌造句 → 评分 → 造成伤害/效果
    ↓                                              ↓
 遭受敌人攻击 ←←←←←← 敌人回合 ←←←←←←← 回合结束
    ↓
 死亡 / 进入下一战
```

**每回合流程：**
1. 从牌库抽 5-7 张词语卡
2. 在有限回合内组合出句子
3. 系统评分 → 转化为伤害/治疗/护盾等效果
4. 未使用的卡牌保留或弃置

### 3.2 卡牌系统详细设计

#### 基础卡牌（白色/灰色）
```
┌─────────────┐
│ 🔵 名词     │
│    「猫」    │
│             │
│ 基础分: 2   │
│ 搭配: 动物类 │
│ 特殊: 无    │
└─────────────┘
```

#### 进阶卡牌（蓝色/紫色）
```
┌─────────────┐
│ 🟣 状语     │
│ 「轻轻地」  │
│             │
│ 修饰倍率: ×1.2│
│ 搭配: 动作类  │
│ 特殊: +柔化效果│
└─────────────┘
```

#### 传说卡牌（金色）
```
┌─────────────┐
│ 🔶 成语碎片  │
│   「卧薪」   │
│             │
│ 单独: 3分    │
│ 组合「尝胆」: │
│ 触发成语奖励  │
│ ×2.5 + 灼烧  │
└─────────────┘
```

### 3.3 协同效应（Synergy）系统

这是游戏深度的关键。参考 Slay the Spire 的协同设计：

#### A. 成语系统（最核心）
- 四张特定单字卡组合 = 成语
- 触发时有华丽特效 + 高额伤害/效果
- 示例：
  - 「破」+「釜」+「沉」+「舟」→ 破釜沉舟：对所有敌人造成巨额伤害，但弃置全部手牌
  - 「画」+「龙」+「点」+「睛」→ 画龙点睛：下一句伤害×3
  - 「守」+「株」+「待」+「兔」→ 守株待兔：本回合不攻击，下回合伤害×4

#### B. 句型协同
- **连动句 build**：多个动词卡连打，每个额外动词 +30% 伤害（连击流）
- **把字句 build**：强制位移 + 控制，把敌人"把"到不利位置
- **被字句 build**：反击流，受到攻击时自动造句反击
- **比较句 build**：对比敌我属性差，差距越大伤害越高
- **排比句 build**：同一结构重复三次，第三次伤害爆发（三段蓄力）

#### C. 意象协同
- 同属一个意象系统的词组合有加成：
  - 🌊 **水系**：江、河、海、流、波、浪、雨、雪 → 流动伤害（DOT）
  - 🔥 **火系**：焰、烧、灼、炎、烈、熔 → 爆发伤害
  - 🌸 **花系**：梅、兰、竹、菊、桃、柳 → 治疗/护盾
  - ⚔️ **武系**：剑、刀、戈、矛、盾、甲 → 直接伤害
  - 📚 **文系**：书、笔、墨、纸、卷、章 → 抽牌/资源

### 3.4 评分公式

```
最终伤害 = (基础语法分 + 词汇品质分) × 修辞倍率 × 主题倍率 × 句长修正

其中：
- 基础语法分 = 成分完整(0-5) + 搭配正确(0-5) + 语序正确(0-3)
- 词汇品质分 = Σ 各卡牌品质值
- 修辞倍率 = 1.0 × (对仗?1.5) × (排比?1.8) × (成语?2.0) × (押韵?1.3)
- 主题倍率 = 1.0 + 0.1×匹配意象数
- 句长修正 = 短句(3字以下)×0.7 / 标准(4-12字)×1.0 / 长句(13+)×1.2（但语法错误风险高）
```

**示例计算：**
> 句子：「烈焰熊熊燃烧着苍茫大地」（火系场景）
> - 语法分：主谓宾完整(5) + 搭配正确(4) + 语序正确(3) = 12
> - 词汇品质：烈焰(4) + 熊熊(3) + 燃烧(3) + 着(1) + 苍茫(4) + 大地(3) = 18
> - 修辞倍率：无特殊修辞 = ×1.0
> - 主题倍率：火系场景，命中「烈焰」「燃烧」「熊熊」= 1.0 + 0.3 = ×1.3
> - 句长修正：10字 = ×1.0
> - **最终 = (12 + 18) × 1.0 × 1.3 × 1.0 = 39 点伤害**

### 3.5 Roguelike 进程设计

#### 地图结构（参考 Slay the Spire）
```
        [诗仙殿]  ← BOSS: 李白（要求押韵对战）
       /    |    \
    [书院] [酒肆] [战场]  ← 精英/事件
     |      |      |
   [村落] [渡口] [山寺]  ← 普通战斗
     \      |      /
      [起点：蒙童]
```

#### 层级设计
| 章节 | 主题 | 解锁语法 | BOSS |
|------|------|---------|------|
| 第一章：蒙学 | 日常生活 | SVO基本句、量词 | 私塾先生（要求造3个语法正确的句子） |
| 第二章：游学 | 山水自然 | 连动句、比较句、定语 | 山中隐士（要求包含自然意象） |
| 第三章：科举 | 治国济世 | 把字句、被字句、排比 | 主考官（要求对仗/排比） |
| 第四章：翰林 | 诗词歌赋 | 成语系统、押韵系统 | 诗仙李白（要求完美押韵+意境） |
| 隐藏章节：超越 | 哲学/禅 | 悖论句、反语、留白 | 庄子（越简洁越强） |

#### 商店/事件系统
- **书斋**（商店）：用金币购买新词卡、升级词卡品质
- **磨墨**（移除）：付费移除低品质卡（"我"→删掉，换"吾"）
- **拜师**（事件）：获得被动能力（如"韵律大师：所有押韵倍率+0.5"）
- **抄书**（事件）：复制一张手牌
- **顿悟**（事件）：随机将一张普通卡升级为成语碎片

#### 遗物系统
| 遗物 | 效果 |
|------|------|
| 🖌️ 湖笔 | 每回合第一个动词卡+2品质 |
| 📜 宣纸 | 手牌上限+2 |
| 🪨 端砚 | 对仗修辞倍率额外+0.5 |
| 💧 徽墨 | 每造出一个合法句子，回复1HP |
| 📖 说文解字 | 可查看牌库中所有成语碎片组合 |
| 🏮 孔明灯 | 每层开始时，预览下3张卡 |
| 🍶 杜康酒 | 修辞倍率永久+0.2，但每回合有10%概率打出错字 |

### 3.6 敌人设计

敌人也用语言主题设计，增加沉浸感：

| 敌人 | 机制 |
|------|------|
| **错别字妖** | 每回合污染一张手牌（变成错字），必须弃置 |
| **语病怪** | 在你的句子中随机插入病句成分 |
| **八股文僵尸** | 每回合释放"模板束缚"，你只能用特定句型 |
| **白话文运动家** | 禁用所有文言词/成语卡 |
| **审查官** | 某些词被"屏蔽"，不能使用 |
| **BOSS: 仓颉** | 每回合拆解你一张卡为偏旁部首，你需要重新组字 |

### 3.7 技术实现建议

#### 语法验证引擎
- **方案A**（推荐入门）：预定义合法句型模板 + 词性标注。词语卡自带词性tag，拖入句子槽位时实时检查是否匹配模板
- **方案B**（进阶）：用 jieba/HanLP 做简单句法分析，判断成分完整性
- **方案C**（联网模式）：调用 LLM API 评分，支持开放式造句

#### 推荐技术栈
- 引擎：Unity 或 Godot（2D卡牌游戏足够）
- 中文处理：本地词典 + 规则引擎（离线可玩）
- 可选AI：GPT/Claude API 用于"自由造句模式"评分

### 3.8 差异化与市场定位

**为什么这个游戏能成立：**

1. **文化共鸣**：中文玩家对成语、诗词有天然情感连接，远超英文拼词游戏
2. **Balatro 验证**：Balatro 证明了"规则简单 + 组合爆炸 + 数值膨胀"的 roguelike 公式在独立游戏市场有巨大需求
3. **空白市场**：目前没有"中文造句 + roguelike deckbuilder"的游戏
4. **教育价值**：可以作为语文学习工具推广（副产品）
5. **AI 时代**：LLM 使得开放式语言评分首次技术可行

**核心风险：**
- 评分系统的"公平感"——玩家觉得自己的句子应该得高分但被判低分
- 内容量——需要大量词语卡和搭配规则
- 平衡性——某些成语组合可能过于强力

---

## 四、总结与下一步建议

### 核心设计原则
1. **Balatro 式评分快感**：基础分 × 倍率 × 倍率 = 数字爆炸的快感
2. **Slay the Spire 式 build 多样性**：不同句型 = 不同流派
3. **文化深度**：成语、诗词、修辞不是装饰，是核心机制
4. **渐进复杂度**：从"主谓宾"开始，逐步解锁高级语法

### 建议的原型验证顺序
1. **纸面原型**：先用实体卡测试"造句 → 评分"的核心乐趣
2. **最小电子原型**：20个名词 + 15个动词 + 10个形容词 + 5个虚词，一个评分引擎，3层关卡
3. **扩展内容**：成语系统、修辞系统、完整4章内容
4. **AI模式**：接入LLM的自由造句模式
